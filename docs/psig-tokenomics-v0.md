# PSI-G Tokenomics v0

## Positioning

PSI-G should not begin as a `meditate-to-earn money` product.

The correct framing is:

- meditate to transform
- complete rituals to progress
- reach milestones to claim rewards

The economy should deepen the spiritual journey, not replace it.

## Core product rule

The spiritual loop must stay primary.

Priority order:

1. 7 Chakra Journey
2. orb growth and awakening
3. missions and rituals
4. gifting and connection
5. points economy
6. optional on-chain claiming

If the user starts optimizing rewards instead of inner practice, the system is failing.

## Recommended economy structure

PSI-G should use a layered economy instead of jumping directly to a tradable token.

### Layer 1: Chakra Progress

Not a currency.

This is the main growth loop:

- 7 days
- 7 chakras
- 7 energy colors
- 1 awakened orb

This is the heart of the product.

### Layer 2: Aura / Resonance Points

This should be the first real economy layer.

Properties:

- off-chain at first
- server-truth, not only local state
- non-financial framing
- used for unlocking, rituals, visuals, and participation

Good candidate names:

- Aura
- Resonance
- Prana
- Light
- Soul Seeds

Recommended default:

- `Resonance`

Reason:

- it feels spiritual
- it connects well with meditation, gifting, and sacred geometry
- it does not sound like a speculative coin

### Layer 3: On-chain Claim Layer

Only introduce this after the product loop is healthy.

Use this for:

- milestone claims
- seasonal badges
- ritual completions
- community events

Do not begin with a fully tradable reward token.

### Layer 4: Optional Future Token

Only consider a true token if:

- retention is strong
- sinks are already proven
- anti-farm systems are working
- claim logic is trusted
- World Mini App review constraints are understood

## What users should earn

Users should not earn financial value every session.

They should earn:

- progress
- meaning
- identity
- ritual access
- rare unlock opportunities

## Earn rules

### Primary earn rules

1. Complete a daily chakra meditation
   - grants a small amount of Resonance
2. Complete the full 7-day chakra journey
   - grants a major milestone reward
3. Complete weekly or seasonal rituals
   - grants medium milestone rewards
4. Complete meaningful gifting / blessing interactions
   - grants small social rewards
5. Reach streak milestones
   - grants milestone rewards, not linear minute rewards

### Best reward types

- Resonance points
- shape unlock fragments
- temporary ritual access
- rare ambient sound trials
- blessing effects
- aura cosmetics

## Spend rules

Resonance must have real sinks.

Recommended spend categories:

1. Unlock limited-time sacred forms
2. Activate special rituals or temple events
3. Enhance gifting effects
4. Open premium-style ceremonial experiences
5. Enter community events or story arcs

### Good examples

- spend Resonance to unlock a 24-hour `Halo Bloom` trial
- spend Resonance to activate a full moon ritual
- spend Resonance to send a high-tier blessing
- spend Resonance to open an oracle reading
- spend Resonance to craft a rare aura effect

### Bad examples

- cash out directly
- instant permanent unlock of everything
- boost your APY
- increase future token yield

## Anti-farm rules

This is critical.

### Never do

- pay users every time they meditate by the minute
- reward idle playback linearly
- let users grind infinite value from repeated short sessions
- give large rewards for simple daily check-in alone
- make VIP a multiplier on financial rewards

### Required anti-farm protections

1. milestone claim model instead of constant micro-drop
2. daily / weekly / monthly claim caps
3. verified-human gating for high-value claims
4. server-side progress ledger
5. suspicious repetition detection for gifting and claim loops

### High-risk abuse patterns

- self-gifting with multiple accounts
- repeatedly starting short sessions
- farming ambient sound switches or page opens
- reinstall / reset abuse if rewards are local only

## VIP relationship

VIP should not be a yield product.

VIP should sell:

- full ambient sound access
- 1 awakened orb
- premium starter forms
- lower daily meditation minutes
- lower mission thresholds

VIP should not sell:

- financial return
- more token yield
- special investment access
- better cashout

### Correct relationship between VIP and Resonance

VIP may:

- lower effort thresholds
- unlock more ways to spend Resonance
- unlock premium rituals where Resonance is used

VIP must not:

- multiply claimable financial rewards
- create a pay-to-farm economy

## World Mini App constraints

PSI-G must stay inside World review-safe boundaries.

### Must avoid

- token pre-sales
- membership tied to yield or APY
- misleading value language
- vague claims about redeemability
- financial framing for spiritual progress

### Must make clear

- what the reward is
- whether it is on-chain or off-chain
- whether it is transferable
- whether it expires
- what it can be used for

## Suggested rollout

### Phase 1: Off-chain Resonance economy

Build first:

- server-truth Resonance balance
- milestone-based earning
- spend sinks inside rituals / visuals / gifting
- no tradable token yet

### Phase 2: On-chain milestone claims

Then add:

- claimable proof for big milestones
- verified-human claim gating
- capped seasonal distribution

### Phase 3: Optional token decision

Only then decide:

- whether PSI-G needs a token at all
- whether the token is governance, utility, or ceremonial

The correct answer may still be:

- keep Resonance off-chain
- use on-chain only for achievements and limited claims

## Final recommendation

PSI-G should not launch with:

- meditate-to-earn token payouts
- highly financial tokenomics
- a yield-like membership model

PSI-G should launch with:

- chakra journey progression
- Resonance as a spiritual utility point
- meaningful sinks
- milestone-based rewards
- optional future on-chain claims
