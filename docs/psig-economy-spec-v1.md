# PSI-G Economy Spec v1

## One-line positioning

PSI-G should use a dual-layer economy:

- `Resonance`
  - off-chain spiritual energy points used every day inside the app
- `PSI`
  - a later on-chain ecosystem token used only after the product loop is healthy

The goal is not `meditate to earn money`.

The goal is:

- meditate to awaken
- complete rituals to earn status and utility
- spend energy inside the ecosystem
- eventually claim selected value on-chain

## Why a dual-layer economy

If PSI-G launches directly as a tradable token app, 3 bad things happen fast:

1. users optimize extraction instead of inner practice
2. abuse and bot behavior arrive before real retention exists
3. World review and policy risk go up immediately

So the correct order is:

1. spiritual loop
2. mission loop
3. off-chain utility economy
4. claim layer
5. optional on-chain token expansion

## Layer A: Resonance

`Resonance` is the app's main daily reward.

It is:

- off-chain
- server-truth
- non-transferable at first
- earned from meaningful progress
- spent on rituals, forms, access, and expression

### Why `Resonance`

It matches the app's world:

- chakras
- sound
- sacred geometry
- gifting
- energy cultivation

It sounds spiritual, not speculative.

## Layer B: PSI token

`PSI` is the future ecosystem token.

It should not be the first reward users see every day.

Instead it should be introduced later as:

- ecosystem utility
- governance or participation layer
- milestone claim layer
- seasonal or ceremonial unlock layer

It can live on World Chain as an ERC-20 once the app proves:

- retention
- demand
- real token sinks
- anti-farm controls
- review-safe positioning

## Product rule

The economy must support the spiritual journey, not replace it.

Priority order:

1. 7 Chakra Journey
2. orb growth
3. awakening
4. missions and rituals
5. gifting and social connection
6. Resonance
7. PSI token

If users start thinking:

- "what is the fastest way to grind"

instead of:

- "what is my next ritual"

the system is broken.

## What to reward

Reward:

- consistency
- ritual completion
- meaningful exploration
- gifting and connection
- milestone completion

Do not reward:

- idle minutes forever
- repeated low-effort taps
- page opens
- sound switching abuse
- VIP simply multiplying raw earnings

## Recommended Resonance earn rules

### Daily base loop

1. Daily check-in
   - `+1 Resonance`
2. Complete the first meditation of the day
   - `+3 Resonance`
3. Complete today's chakra ritual
   - `+5 Resonance`

### Progress rewards

1. Reach a 3-day streak
   - `+6 Resonance`
2. Reach a 7-day streak
   - `+12 Resonance`
3. Complete the full 7 Chakra Journey
   - `+25 Resonance`
4. Awaken your main orb
   - `+20 Resonance`

### Exploration rewards

1. Try a new meditation theme
   - `+2 Resonance`
2. Complete the first AI-generated meditation
   - `+4 Resonance`
3. Generate your first AI meditation
   - `+3 Resonance`

### Social rewards

1. Send your first blessing of the day
   - `+2 Resonance`
2. Receive a blessing
   - `+1 Resonance`
3. Complete a weekly gifting milestone
   - `+5 Resonance`

## Daily caps

To avoid farming, cap daily earnings.

Recommended starting cap:

- free user daily cap: `12 Resonance`
- VIP daily cap: `12 Resonance`

Important:

- VIP should unlock more value and smoother progression
- VIP should not directly print more currency

## Resonance sinks

Resonance only works if people can spend it in desirable ways.

### Best sink categories

1. temporary shape access
2. aura or particle effects
3. ritual event entry
4. oracle or temple experiences
5. blessing upgrades
6. AI content unlocks
7. cosmetic garden prestige

### Suggested starting sink table

1. Rare shape 24-hour trial
   - `20 Resonance`
2. Full moon ritual entry
   - `12 Resonance`
3. Premium blessing effect
   - `5 Resonance`
4. AI ceremonial meditation generation
   - `8 Resonance`
5. Special orb aura colorway
   - `15 Resonance`
6. Expanded blessing gallery slot
   - `10 Resonance`

## VIP relationship

VIP should remain a premium experience product, not a yield product.

VIP gives:

- all ambient sounds
- 1 gifted awakened orb for instant exploration
- premium starter shapes
- lower effort thresholds for some missions
- easier access to certain Resonance sinks

VIP does not give:

- more daily currency cap
- higher raw token yield
- APY
- "investment return"

### Correct VIP + Resonance relationship

VIP may:

- reduce grind
- unlock better ways to spend
- open premium rituals

VIP must not:

- multiply financial rewards
- turn the app into pay-to-farm

## PSI token design direction

When PSI is introduced, it should begin as a utility token.

### PSI uses

1. governance for seasonal ritual directions
2. premium event entry
3. staking for cosmetic prestige, not cash yield promises
4. access to advanced creator or AI tools
5. symbolic long-term alignment with the ecosystem

### PSI should not launch as

1. meditate-to-earn cash token
2. presale product inside the mini app
3. "buy VIP to farm more PSI"
4. a vague speculative asset with no sinks

## Suggested rollout

### Phase 1: Resonance only

Build first:

- server-truth balance
- append-only ledger
- mission-based earning
- sink-based spending
- anti-farm controls

No tradable token yet.

### Phase 2: Claim layer

Add:

- big milestone claims
- seasonal rewards
- ceremony badges
- soulbound style recognition

This is where on-chain identity starts to make sense.

### Phase 3: PSI token soft launch

Only after Phase 1 and 2 prove healthy:

- introduce PSI on World Chain
- keep issuance limited
- keep sinks stronger than hype
- do not center the app around speculation

### Phase 4: Ecosystem expansion

Only later:

- creator economy
- community ritual voting
- temple events
- advanced AI experiences
- ecosystem-level token use

## Anti-farm rules

This is mandatory.

### Never do

1. pay linearly per meditation minute forever
2. reward every app open
3. allow self-gifting to create currency loops
4. let VIP boost financial return
5. make local-only rewards claimable as truth

### Must do

1. server-side reward ledger
2. unique claim ids
3. daily caps
4. suspicious activity checks
5. self-send exclusions
6. repeatable reward cooldowns

## Investor-facing story

If you eventually talk to investors, the strongest story is not:

- "we have a token"

The strongest story is:

- PSI-G has a spiritually differentiated retention loop
- the orb system creates symbolic identity
- AI meditation deepens content supply
- gifting builds social ritual
- Resonance proves utility demand
- PSI later becomes the ecosystem coordination layer

That story is much stronger than leading with speculation.

## World safety notes

PSI-G should stay review-safe by avoiding:

- token pre-sale inside the mini app
- yield language
- APY framing
- membership tied to higher financial return
- misleading redemption promises

Every reward should clearly state:

- off-chain or on-chain
- transferable or non-transferable
- permanent or temporary
- what it can actually be used for

2. connect daily check-in, daily meditation, chakra completion
3. add 3 to 5 real Resonance sinks
4. add weekly summary and spend surfaces
